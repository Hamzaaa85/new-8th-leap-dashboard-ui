# Project Guidelines

This dashboard should be built with a scalable, modular, clean, and production-ready architecture.

Future work must follow these principles:

- Keep components small, reusable, and easy to maintain.
- Organize code by clear feature and responsibility boundaries.
- Use clean naming, consistent formatting, and readable structure.
- Follow production best practices for UI, state, API integration, validation, error handling, and performance.
- Keep the codebase easy to extend as new dashboard screens, APIs, and business logic are added.
- Match provided UI design screens as closely as possible, with pixel-perfect attention where required.
